 Check Security Group Settings (If Not Accessible)
If the website is not accessible, you might need to ensure that the correct port (8000 in this case) is open in your EC2 Security Group.

Go to AWS Management Console:

Navigate to EC2 > Instances, find your instance, and click on Security Groups.
Edit Inbound Rules:

Click on the associated Security Group.
Go to the Inbound Rules tab, and click Edit Inbound Rules.
Add a rule that allows HTTP traffic on port 8000:
Type: Custom TCP Rule
Protocol: TCP
Port Range: 8000
Source: Anywhere (or your IP address for more security)
Save Changes and try accessing the website again.
